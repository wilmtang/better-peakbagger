General

Enable experimental 3D map Experimental Shows a 3D terrain option on ascent maps, Full Screen GPS maps, and Peak pages. Loading it requests elevation tiles from open-source project Mapterhorn; choosing OSM Vector requests map tiles from OpenFreeMap. Those services receive the viewed map area and request metadata.

Activity creation

GPX capture

Retain GPX waypoints Include waypoint names and coordinates in the GPX sent to Peakbagger. Other waypoint details remain excluded.
Ascent details Fill elevation, distance, time, and gain for the route up and down, plus day-by-day statistics for overnight ascents.
Trip info for multiple peaks Create one Peakbagger trip, named after the activity, with peaks sequenced in track order.
Wilderness nights for overnight ascents Fill nights out from the GPX date span on every overnight draft and reveal Peakbagger’s day-by-day table.
External trip report link Put the captured Garmin or Strava link into Peakbagger’s “URL Link to External Trip Report” field when it is empty.

Trip report editor

Trip report editor Replace the ascent form’s trip report box with a rich text and Markdown editor that saves in Peakbagger’s own format, with drafts kept on this device. “Plain” mode inside the editor is the original textarea.
Credit Better Peakbagger in trip reports Add a small, editable store link to blank trip reports. You can change or remove it before saving.

Map & GPX chart

GPX chart

Map

Map route appearance Style the ascent-page GPX overlay. Route width also applies to native GPS tracks on Full Screen Maps; their original colors, hover highlights, and click details remain unchanged.
Remember map layer Load the layer you last selected on an ascent map. Turning this off forgets the saved layer.
3D terrain cache Keep elevation tiles on this device for faster repeat views. Your browser may remove them when space is tight. Checking current cache…
Map viewport By default, ascent maps use Peakbagger’s original 450 × 450 px size. Drag the lower-right corner up to the page width; changes apply to every ascent page.

Ascent beta filter

“Has beta” means What the ascent-list Has beta chip counts. An ascent qualifies if it has at least one of the checked signals.
Newest ascents first Open ascent lists sorted by date, newest first, when the page arrives in its default oldest-first order. A sort you choose by clicking a column always wins.

GitHub trip backup

Back up ascents to GitHub Back up saved ascents individually, or open My Ascents to archive every ascent from every year — the trip report as Markdown, every field as JSON, and Peakbagger’s stored GPX track — to one GitHub repository you choose.
